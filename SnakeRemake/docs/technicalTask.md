# Техническое задание для игры "SnakeRemake"

---

## 1. Введение
**1.1. Наименование проекта**  
SnakeRemake - современная версия классической игры "Змейка" с ботами-соперниками.

**1.2. Назначение**  
Создание динамичной игры со стратегическими элементами, где игрок соревнуется с ИИ-змеями за ресурсы.

---

## 2. Требования к функциональности
**2.1. Основные функции:**
- Управление змейкой игрока (стрелки на клавиатуре)
- Генерация еды на игровом поле
- Два типа ботов:
  - Атакующие еду (желтые)
  - Атакующие игрока (красные)
- Система очков:
  - +10 очков за сбор еды
  - +1 очко за уничтожение бота
- Периодический спавн новых ботов
- Игровые меню:
  - Стартовое меню
  - Игровой интерфейс
  - Меню поражения

**2.2. Логика ИИ:**
- Боты выбирают кратчайший путь к цели
- Избегание столкновений с другими ботами
- Условия удаления бота: столкновение с игроком или другим ботом

**2.3. Условия завершения игры:**
- Столкновение с границами поля
- Столкновение с собственным телом
- Столкновение с ботом

---

## 3. Технические требования
**3.1. Платформа и окружение:**
- Операционная система: Windows
- Технологии: WPF (.NET Framework), C#
- Разрешение окна: 650x520 пикселей
- Размер игрового поля: 600x400 пикселей

**3.2. Производительность:**
- Частота обновления: 200 мс/кадр
- Поддержка до 20 одновременно активных ботов

**3.3. Архитектура:**
- Игровое поле: сетка 60x40 ячеек (размер ячейки: 10x10 пикселей)
- Основные сущности: игрок, боты, еда
- Отрисовка через Canvas

---

## 4. Интерфейс пользователя
**4.1. Визуальные элементы:**
- Игрок:
  - Голова: светло-зеленый (Lime)
  - Тело: зеленый (Green)
- Боты:
  - Атакующие еду: 
    - Голова: желтый (Yellow)
    - Тело: оранжевый (DarkOrange)
  - Атакующие игрока:
    - Голова: красный (Red)
    - Тело: темно-красный (DarkRed)
- Еда: красный квадрат (Red)
- Текстовые элементы: счет игрока

**4.2. Меню:**
- Стартовое меню:
  - Заголовок "SnakeRemake"
  - Кнопки: "Start Game", "Exit"
- Меню поражения:
  - Заголовок "Game Over"
  - Итоговый счет
  - Кнопки: "Restart", "Back"

---

## 5. Тестирование
**5.1. Ключевые сценарии:**
- Движение игрока и ботов
- Генерация еды
- Обработка столкновений
- Начисление очков
- Спавн ботов

**5.2. Инструменты тестирования:**
- Модульные тесты (проект GameTests)
- Тестирование методов:
  - MoveSnake()

  - SpawnFood()

## 7. Требования к разработке
**7.1. Стандарты кодирования:**
- Соблюдение принципов ООП
- Комментирование сложных алгоритмов
- Использование модификатора internal для тестируемых методов

**7.2. Сборка и развертывание:**
- Целевая платформа: .NET Framework 4.7+
- Отсутствие внешних зависимостей
- Простая установка через исполняемый файл

---

## 8. Риски
**8.1. Технические риски:**
- Снижение производительности при большом количестве ботов (более 25)
- Возможные ошибки в логике ИИ при сложных сценариях преследования
- Проблемы с определением коллизий в угловых случаях

**8.2. Риски установки и развертывания:**
- Отсутствие необходимой версии .NET Framework на целевых машинах
- Проблемы с разрешениями экрана на разных устройствах
- Конфликты с системными библиотеками

**8.3. Меры по снижению рисков:**
- Оптимизация алгоритмов обработки коллизий
- Добавление случайности в поведение ИИ
- Четкая документация системных требований
- Предоставение инсталлятора с проверкой зависимостей
